Implementação: Definir uma Métrica de Desempenho¶

difcil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso geralmente feito utilizando algum tipo de mtrica de desempenho, atravs do clculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida til. Para este projeto, voc ir calcular o coeficiente de determinao, R2, para quantificar o desempenho do seu modelo. O coeficiente de determinao uma estatstica til no campo de anlise de regresso uma vez que descreve o quo "bom" a capacidade do modelo em fazer estimativas.

Os valores para R2 tm um alcance de 0 a 1, que captura a porcentagem da correlao ao quadrado entre a estimativa e o valor atual da varivel alvo. Um modelo R2 de valor 0 sempre falha ao estimar a varivel alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a varivel alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da varivel alvo (ao utilizar o modelo) que pode ser explicada pelos atributos. Um modelo pode dar tambm um R2 negativo, que indica que o modelo no melhor do que aquele que estima ingenuamente a mdia da varivel alvo.

Para a funo performance_metric na clula de cdigo abaixo, voc ir precisar implementar o seguinte:

  • Utilizar o r2_score do sklearn.metrics para executar um clculo de desempenho entre y_true e y_predict.
  • Atribuir a pontuao do desempenho para a varivel score.

from sklearn.metrics import r2_score

def performance_metric(y_true, y_predict):
    """ Calcular e retornar a pontuao de desempenho entre 
        valores reais e estimados baseado na mtrica escolhida. """
    
    # Calcular a pontuao de desempenho entre 'y_true' e 'y_predict'
    score = r2_score(y_true, y_predict)
    
    # Devolver a pontuao
    return score

Dica: Quando explicar a tcnica de busca em matriz, tenha certeza que voc explicou o motivo dela ser usada, o que a 'matriz' significa nesse caso e qual o objetivo da tcnica. Para ter uma resposta mais slida, voc pode tambm dar exemplo de um parmetro em um modelo que pode ser otimizado usando essa tcnica.

Resposta: O grid search uma tcnica que visa identificar quais parmetros de um algoritmo de aprendizagem so ideais para o treinamento sobre um determinado conjunto de dados. A ideia de busca em matriz vem do fato de que essa tcnica busca a melhor combinao de valores de parmetros, sendo cada parmetro equivalente uma dimenso de uma matriz.

Por exemplo, para um algoritmo de rvore de deciso, podemos especificar os parmetros de profundidade mxima e de nmero mnimo de exemplos que justificam um n. Para usar o grid search nesse cenrio, escolhemos uma srie de valores interessantes para os dois parmetros e validamos exaustivamente modelos treinados com cada combinao de valores. Ento, so selecionados os valores dos parmetros que geraram o modelo com o melhor desempenho de validao.

Questão 8 - Validação Cruzada¶

  • O que a tcnica de treinamento de validao-cruzada k-fold?
  • Quais benefcios essa tcnica proporciona para busca em matriz ao otimizar um modelo?

Dica: Lembre-se de expllicar o que significa o 'k' da validao-cruzada k-fold, como a base de dados dividida e quantas vezes ela executada.

Assim como h um raciocnio por trs de utilizar um conjunto de teste, o que poderia dar errado ao utilizar busca em matriz sem um conjunto de validao cruzada? Voc pode utilizar a documentao para basear sua resposta.

Resposta: O k-fold cross validation uma tcnica de separao do conjunto de dados. Isolado o conjunto de teste, essa tcnica divide o restante dos dados em k parties. Isso permite executar o treinamento e a validao de k modelos, cada vez utilizando uma das parties como conjunto de validao e as demais para treinamento. Isso gera resultados para diferentes combinaes do conjunto de dados e permite calcular estatsticas mais confiveis sobre a performance do modelo.

Essa tcnica recomendada porque, quanto mais dividimos fixamente o conjunto de dados para diferentes propsitos, mais enviesado o modelo resultante se torna. Quando falamos de grid search, por exemplo, se utilizamos sempre os mesmos 10% dos dados como conjunto de validao, possvel que os parmetros selecionados sejam adequados demais para aqueles 10%. Por isso, recomenda-se, por exemplo, utilizar k-fold cross validation em cada "clula" durante o grid search. Embora seja computacionalmente mais caro, permite uma imparcialidade maior na seleo dos parmetros.

from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.model_selection import GridSearchCV

def fit_model(X, y):
    """ Desempenhar busca em matriz sobre o parmetro 'max_depth' para uma 
        rvore de deciso de regresso treinada nos dados de entrada [X, y]. """
    
    # Gerar conjuntos de validao-cruzada para o treinamento de dados
    cv_sets = ShuffleSplit(n_splits=10, test_size = 0.20, random_state=0)

    # TODO: Gerar uma rvore de deciso de regresso de objeto
    regressor = DecisionTreeRegressor()

    # TODO: Gerar um dicionrio para o parmetro 'max_depth' com um alcance de 1 a 10
    params = {
        'max_depth': list(range(1, 11))
    }

    # TODO: Transformar 'performance_metric' em uma funo de pontuao utilizando 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Gerar o objeto de busca em matriz
    grid = GridSearchCV(
        regressor,
        params,
        scoring = scoring_fnc,
        cv =cv_sets
    )

    # Ajustar o objeto de busca em matriz com os dados para calcular o modelo timo
    grid = grid.fit(X, y)

    # Devolver o modelo timo depois de realizar o ajuste dos dados
    return grid.best_estimator_
